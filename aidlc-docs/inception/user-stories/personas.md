# User Personas

## Persona 1: 식당 방문 고객 (Customer)

| 항목 | 내용 |
|------|------|
| **이름** | 김민수 (대표 페르소나) |
| **역할** | 식당 방문 고객 |
| **환경** | 테이블에 설치된 태블릿 사용 |
| **기술 수준** | 다양 (스마트폰 사용 가능 수준이면 충분) |

### 목표 (Goals)
- 빠르고 쉽게 메뉴를 확인하고 주문하기
- 직원 호출 없이 편하게 추가 주문하기
- 현재 주문 상태를 실시간으로 확인하기

### 동기 (Motivations)
- 대기 시간 최소화
- 메뉴 사진과 설명을 보고 신중하게 선택
- 주문 실수 방지 (장바구니에서 확인 후 확정)

### 불만사항 (Pain Points)
- 직원을 불러도 오래 기다려야 하는 상황
- 메뉴판이 복잡하거나 가격이 잘 안 보이는 경우
- 주문이 제대로 들어갔는지 불안한 경우

### 시나리오
1. 테이블에 앉으면 태블릿이 자동으로 메뉴 화면 표시
2. 카테고리별로 메뉴를 탐색하며 원하는 메뉴 선택
3. 장바구니에 담고 수량 조절 후 주문 확정
4. 주문 상태(대기중/준비중/완료)를 확인
5. 추가 주문이 필요하면 같은 과정 반복

## Persona 2: 매장 관리자 (Admin)

| 항목 | 내용 |
|------|------|
| **이름** | 박서연 (대표 페르소나) |
| **역할** | 매장 사장님 / 운영 관리자 |
| **환경** | PC 또는 태블릿으로 관리자 페이지 접속 |
| **기술 수준** | 기본적인 웹 서비스 사용 가능 |

### 목표 (Goals)
- 실시간으로 들어오는 주문을 빠르게 확인하고 처리
- 메뉴를 쉽게 등록/수정/삭제
- 테이블별 주문 현황을 한눈에 파악
- 영업 종료 시 테이블 정리(이용 완료 처리)

### 동기 (Motivations)
- 주문 누락 방지
- 효율적인 매장 운영
- 고객 대기 시간 최소화

### 불만사항 (Pain Points)
- 주문이 밀릴 때 놓치는 경우
- 테이블별 주문 현황 파악이 어려운 경우
- 메뉴 변경 시 번거로운 작업

### 시나리오
1. 영업 시작 시 로그인 후 주문 대시보드 확인
2. 신규 주문 알림 수신 → 주문 상태를 "준비중"으로 변경
3. 음식 완료 시 "완료"로 상태 변경
4. 필요 시 메뉴 추가/수정/가격 변경
5. 고객 퇴장 시 테이블 이용 완료 처리
6. 과거 주문 내역 확인 (필요 시)
